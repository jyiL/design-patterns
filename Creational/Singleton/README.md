### 单例
单一的实例(mysqldb类),节省对象资源

1、防止用户通过new关键字无限实例化新对象
将构造函数私有化,就无法在类外面new一个新的对象

2、 增加一个静态的公共方法进入类的内部

3、 增加一个私有的静态属性

4、 将克隆魔术方法私有化

`总结: 单例模式的最经典做法,三私一公`

* 私有化构造方法: 防止使用 new 创建多个实例
* 公共的静态方法: 进入类的内部
* 私有化静态属性: 防止直接访问存储实例的属性
* 私有化克隆方法: 防止 clone 多个实例

5、将__wakeup方法私有化

`总结: 四私一公`
* 私有化构造方法: 防止使用 new 创建多个实例
* 公共的静态方法: 进入类的内部
* 私有化静态属性: 防止直接访问存储实例的属性
* 私有化克隆方法: 防止 clone 多个实例
* 私有化__wakeup方法: 防止反序列化
